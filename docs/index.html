Documentation

This documentation describes how to use the Brain Entropy Index (BEI) Toolbox to load datasets, preprocess signals, compute energy-based metrics, visualize subject-level and population-level behavior, and evaluate predictive models. The content is written to map directly to the BEI Toolbox GUI workflow.

Core idea: BEI Toolbox separates pointer-based data organization (Datasets, Studies) from processing state (registries) and from derived products and outputs (PDI, PDO, testing results, and plots).

Terminology used in this documentation

Term Meaning in BEI Toolbox
Dataset A lightweight definition that points to where the raw data live and how they should be interpreted (modality and diagnosis). The dataset file is saved under datasets_store/<dataset-modality-diagnosis>/. Raw data are not replicated.
Study A user-defined grouping of two or more datasets for population-level analysis and model evaluation. The study file is saved under study_store/<study-name>/. It stores pointers to datasets and does not replicate raw data.
PDI Product Data Input created after successful preprocessing of a dataset and stored in the dataset PDI/ directory. The PDI contains a patient_data structure with one entry per participant.
PDO Product Data Output created after successful energy-metrics calculations for a study and stored in the study PDO/ directory. The PDO contains an outputest structure with one entry per participant.
Testing results Output of model testing stored in testing_store/ as BEI_TestResults_<StudyName>.mat. Contains the BEI_TestResults table with labels, predictions, BEI scores, metrics, and evaluation summaries.
Registry Persistent state stored in the BEI Toolbox memory that tracks the state of each dataset and study and which menus should be enabled (preprocessing, energy metrics, testing).
Tip: In the website repo, keep documentation pages under /docs/. In the MATLAB distribution, you can keep a copy inside the toolbox so the docs match the installed version.
