Studies

A study is a population-level construct: it specifies a cohort composed of two or more datasets and defines how subjects are pooled for group statistics, energy distribution modeling, and downstream classification. Studies are the unit at which the toolbox produces reproducible population artifacts and summarizes results across diagnoses and modalities.

Why studies exist

Most BEI properties are defined on distributions and network structure across subjects, not on a single recording. Studies provide the statistical container needed to estimate equal energy allocation, coupling proxies, and entropy-derived indices under consistent assumptions.

Definition

A study is a study-scoped container that references two or more datasets and produces population-level outputs. Studies are created from the registered datasets in the current session and are saved under studies_store.

When you define a study, you select one or more datasets and provide a short study name. The name must be 1 to 12 characters and contain only letters and digits. The toolbox then creates a dedicated study folder and three subfolders for study artifacts: SS/ (study definition), PDI/ (study population snapshots), and PDO/ (study-level outputs). 

studies_store/
  <STUDY_NAME>/
    SS/<STUDY_NAME>_BEI_ST.mat
    PDI/
    PDO/

The study definition file (*_BEI_ST.mat) contains a study struct with pointers to the selected datasets (study.refs), plus summary metadata such as total subject count, modalities, and diagnoses.

To load an existing study, use File → Load Study and select a *_BEI_ST.mat file from studies_store. Loading a study activates it for the Plot, Energy Metrics, and Testing menus.

Cohort assembly

Study cohort assembly is performed from dataset-level preprocessing outputs. For each selected dataset, the toolbox locates the dataset PDI/ directory and loads the most recent PDI_*.mat file. The patient_data entries from all selected datasets are concatenated into a single in-memory cohort.

After dataset subjects are merged, the toolbox may append internal Healthy Control subjects based on the modalities present in the study. The final concatenated patient_data struct array is saved as a versioned study-level PDI file under the study PDI/ directory.

The version number is incremented automatically so that prior cohort snapshots remain available for traceability.

PDI outputs

The primary persistent artifact at the study level is a PDI file (Population Data Interface). A PDI is a MATLAB .mat file that contains the concatenated patient_data struct array for all subjects included in the study. When a study is defined, the toolbox loads the most recent dataset-level PDI from each selected dataset, concatenates them, optionally appends internal healthy control subjects (one internal cohort per modality), and writes a versioned study-level PDI under the study's PDI/ directory.

BEI_ToolBox/
  studies_store/
    <STUDY_NAME>/
      SS/
        <STUDY_NAME>_BEI_ST.mat
      PDI/
        PDI_<NDIAG>D_<STUDY_NAME>_v<N>.mat
      PDO/
        PDO_<NDIAG>D_<STUDY_NAME>_v<N>.mat

A PDI should be treated as a versioned snapshot. Whenever you change study membership or recompute population features, write a new PDI and keep the prior one for traceability.

Typical PDI content

% Illustrative, exact fields may vary by toolbox version
PDI.patient_data          % array of patient_data structs (possibly reduced)
PDI.study_name            % string
PDI.created_at            % timestamp
PDI.toolbox_version       % version string
PDI.datasets_included     % list of dataset names
PDI.n_subjects            % resolved subject count
PDI.notes                 % optional
Avoid accidental empty entries

When building the cohort, filter out any uninitialized structs so downstream tables do not break due to size mismatches. A simple rule is to keep only entries that contain required fields such as patient_code and diagnosis.

Internal controls

Studies can automatically include an internal Healthy Control reference cohort, even if those subjects are not part of the datasets you selected in the study picker. This is used to ensure a consistent baseline for population plots and model evaluation across sessions.

Internal Healthy Control subjects are appended during study definition on a per-modality basis. The toolbox inspects the modalities present in the selected datasets and, for each modality, loads the most recent internal PDI file and concatenates its patient_data with the study cohort before writing the study-level PDI. 

datasets_store/
  BEI_EEG_HC/PDI/PDI_1D_*.mat
  BEI_fMRI_HC/PDI/PDI_1D_*.mat
  BEI_MEG_HC/PDI/PDI_1D_*.mat

If a modality has no internal cohort available on disk, the study is still created, but no internal subjects are appended for that modality.

Audits and quality gates

Before computing energy metrics or training models, studies should pass basic quality checks:

  • Label completeness: every subject has diagnosis and modality labels.
  • Signal integrity: no empty time series, consistent dimensions, consistent units.
  • Cohort balance visibility: subject counts by diagnosis and modality are logged and displayed.
  • Reproducible provenance: toolbox version and preprocessing recipes are discoverable from the PDI.
