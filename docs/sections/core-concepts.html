Core concepts

Dataset

In BEI Toolbox, a dataset is a collection of participants brain imaging recordings from one source (center) that are processed under a consistent set of assumptions/parameters. A dataset typically corresponds to a directory tree containing participant folders or files, and the dataset definition stores pointers to those locations, declared modality (EEG, iEEG, MEG, or fMRI) and unique diagnosis.

  • Datasets are the unit of preprocessing and subject-level plots.
  • Datasets are loaded without preprocessing.
  • Datasets do not require other datasets to exist.

Study

A study is a user-defined grouping of two or more datasets that enables population-level statistics and consistent cohort comparisons. Studies are designed for multi-diagnosis and multi-modality analysis.

  • Studies are the unit of population-level plots and model evaluation.
  • Studies contain references to datasets rather than duplicating raw data.
  • Studies include “internal datasets” that are synthesized from existing processed subjects.
Rule of thumb: If you want group statistics, build a study. If you want to inspect individual subjects, use datasets.

Dataset storage and non-replication

Datasets are saved inside datasets_store/ under a folder named <dataset-modality-diagnosis>. The dataset definition is lightweight and stores pointers to where raw data are located. BEI Toolbox does not copy or replicate the raw recordings.

PDI (Product Data Input)

When preprocessing completes successfully, BEI Toolbox writes a Product Data Input file inside the dataset PDI/ directory. The PDI contains a patient_data structure with one entry per participant.

patient_data entry fields

  • participant_code
  • diagnosis
  • modality
  • dataset
  • brain imaging array (for example eegarray, megarray, or fMRIarray)

Study storage and non-replication

Studies are saved inside study_store/ under a folder named <study-name>. A study stores references to its constituent datasets. Raw data are not replicated.

Energy metrics and BEI properties

BEI Toolbox computes a family of energy-based metrics from participant PDIs of a study and aggregates them for population analysis. Different metrics support different statistical properties, for example equal allocation of energy across regions, coupling behavior, and variance structure across subjects or groups.

PDO (Product Data Output)

When energy metrics are computed successfully for a study, BEI Toolbox writes a Product Data Output file inside the study PDO/ directory. The PDO contains an outputest structure with one entry per participant.

outputest entry fields

  • participant_code
  • diagnosis
  • dataset
  • sample mean
  • sample variance
  • log-likelihood
  • EED (Equal Energy Distribution)
  • coupling Ratio
  • variance Ratio
  • brain stability
  • modality
Metrics are operational. They are defined so that they can be computed consistently across datasets and modalities, with reproducible preprocessing gates.

Testing results

When a trained model is tested with a selected study, the results are saved in testing_store/ as BEI_TestResults_<StudyName>.mat. The file contains a BEI_TestResults table summarizing study name, model name, diagnosis labels, predicted labels, BEI scores, energy metrics, confusion matrix, performance metrics, and thresholds.

Registries

Registries are stored in the BEItoolbox memory and act as the source of truth for workflow state. They exist to:

  • Track which datasets have been preprocessed.
  • Track which studies have computed energy metrics.
  • Track which studies have been tested with a model.
  • Enable or disable menus so users cannot run analysis steps in an invalid order.
