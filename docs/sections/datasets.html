Datasets

In the BEI Toolbox, a dataset is the smallest top-level unit of organization. A dataset represents a coherent collection of subjects acquired under a shared protocol and modality (EEG, iEEG, MEG, or fMRI) as well as a stable diagnosis label policy. Dataset-level operations are designed to be deterministic: the same raw inputs and the same toolbox version should yield identical subject-level outputs, subject to upstream numerical libraries and platform differences.

Key point

Datasets in the GUI are primarily about discovery, indexing, and subject-level preprocessing. Population-level energy metrics are computed at the study level.

Definition

A dataset is defined by four categories of information:

  • Identity: a stable name and a canonical folder root that points to the raw data organization.
  • Modality: EEG, iEEG, MEG, or fMRI, which determines loaders, units, and labels.
  • Diagnosis policy: mapping from raw metadata to standardized labels used in analysis.
  • Participant index: reproducible subject identifiers and file pointers derived from a directory scan.

Registration and metadata

When you load or define a dataset in the GUI, the toolbox registers it so it can be displayed in menus, preprocessed, and used as a source of subjects for studies. 

% Typical dataset fields (illustrative)
dataset.name               % string, dataset identifier used in the GUI
dataset.modality           % 'EEG' | 'iEEG' | 'MEG' | 'fMRI'
dataset.patient_diagnosis  % diagnosis label for single-diagnosis datasets (optional)
dataset.rootDir            % raw data root directory
dataset.subDirs            % subject folder list or file list

Storage outputs

Dataset definition is saved as a MATLAB .mat file in the dataset’s DS folder. When you run preprocessing, the toolbox writes dataset-scoped PDI outputs in the dataset’s PDI folder. A PDI file contains a patient_data struct (typically an array with one element per subject) produced by the preprocessing pipeline. 

BEI_ToolBox/
  datasets_store/
    <NAME>_<MODALITY>_<DIAGNOSIS>/
      DS/
        <NAME>_<MODALITY>_<DIAGNOSIS>_BEI_DS.mat
      PDI/
        PDI_<timestamp>.mat
      PDO/
        (reserved for outputs written by study-level analyses)

Creating a dataset (File → New Dataset) defines the dataset, creates the DS/PDI/PDO folders under datasets_store, and saves a dataset definition file named <NAME>_<MODALITY>_<DIAGNOSIS>_BEI_DS.mat. Loading a dataset (File → Load Dataset) reloads that definition from disk and registers it in the current GUI session.

PDI content model

A PDI is a MATLAB .mat file containing a patient_data struct (often a struct array with one element per subject). Each entry stores the preprocessed signal matrix and the minimal metadata required for downstream energy and entropy analysis (for example: patient code, modality, diagnosis, dataset identifier, and channel or ROI labels when available).

Units matter

In downstream energy and entropy computations, scale errors can mimic physiological effects. The preprocessing pipeline should enforce consistent units before feature extraction. 

patient_data.BIarray         % [T x C] preprocessed EEG/iEEG/MEG/fMRI time series
patient_data.chan_labels     % [1 x C] channel or ROI labels
patient_data.patient_code    % e.g., 'sub-XXXX'
patient_data.diagnosis       % standardized diagnosis label
patient_data.modality        % 'EEG' | 'MEG' | 'fMRI' | 'iEEG'
patient_data.dataset         % dataset name
patient_data.A               % optional state matrices (if computed)

Recommended practices

  • Keep dataset names stable. Dataset names are used in registry keys and folder paths.
  • Do not mix modalities or diagnoses inside a dataset. Comparisons belong at the study level.
