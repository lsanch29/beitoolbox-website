Menus

This page describes every top menu and submenu in the BEI Toolbox GUI and explains when items become available. Menu availability is driven by dataset and study status fields and by the presence of a loaded model.

File

The File menu manages datasets and studies stored under the toolbox working directories.

  • New Dataset: Define a dataset by selecting a root folder and assigning name, modality, and patient_diagnosis. The definition is saved under datasets_store as a versioned dataset file.
  • Load Dataset: Load a previously saved dataset definition (*_BEI_DS.mat) from datasets_store. Loading refreshes the GUI registry, menus, and the Info Panel.
  • Remove Dataset: Remove a loaded dataset from the current session. This disables dependent tools for that dataset until it is loaded again.
  • New Study: Define a study that references two or more datasets. Studies are stored under studies_store and enable cross dataset population analysis.
  • Load Study: Load a saved study definition from studies_store. Loading refreshes menus and enables study scoped analysis when energy metrics exist.
  • Remove Study: Remove a loaded study from the current session.

Tools

Tools are grouped by analysis stage: preprocessing, energy metrics, plotting, and testing.

Pre-Processing

The Pre-Processing submenu is populated dynamically by buildPreProcessingMenu.m. Each entry represents one loaded dataset.

  • Each dataset entry is labeled as [DATASET]: name – modality – diagnosis: N subjects - status.
  • Selecting a dataset entry runs preprocessing for that dataset via runPreprocessing.
  • If no datasets are loaded, the menu shows (No Datasets/Studies Available) and is disabled.

Energy Metrics

The Energy Metrics submenu is populated by buildEnergyMetricsMenu.m. It can include both datasets and studies.

  • Datasets appear only after they are preprocessed. A dataset must have processingStatus = Preprocessed to be listed.
  • Studies can be listed whenever they exist in the session. Energy metrics can be calculated at study scope to support population analysis.
  • Each entry runs energyMetricsCallback and writes a versioned PDO*_v*.mat file containing outputest.

Testing

The Testing menu is built by buildTestingMenu.m and has two steps: load a model, then test it on an eligible dataset or study.

  • Select Model: Loads a previously trained model into the session. This item is enabled only when at least one eligible dataset or study exists.
  • Test Model: A submenu populated with eligible test sets. It is enabled only after a model is loaded.
  • Eligibility rules: datasets must be preprocessed and must have energy metrics; studies must have energy metrics.
  • Test status for studies: derived from BEI_TestResultsRegistry stored in session appdata, not from files on disk.

Plot

The Plot menu provides plotting utilities. It is organized into Subject-Level, Population-Level, and Model Evaluation sections.

  • Subject-Level → Energy PDF / Cov Matrix: Opens subject level plots driven by the latest PDI output for the selected dataset.
  • Population-Level: Produces cohort wise box plots for selected metrics by reading the latest PDO*_v*.mat file and the outputest struct.
  • Model Evaluation: Provides plots such as Population BEI, ROC and calibration, confusion matrix, clinical zones histogram, and forest plots.

Menu availability rules

Several menus reflect the state of the current session. The most common reasons an item is disabled are listed here.

  • No datasets loaded: preprocessing and dataset dependent plots show placeholder entries and are disabled.
  • Dataset not preprocessed: it will not appear in Energy Metrics and it cannot be used for testing.
  • Energy metrics not calculated: population plots and testing are not available for that dataset or study.
  • No model loaded: Test Model submenu remains disabled even if eligible test sets exist.
