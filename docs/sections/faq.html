FAQ

Why does the toolbox refuse to start?

The toolbox performs dependency checks for required MATLAB products and external toolboxes. If a dependency is missing, it exits early to avoid producing silently invalid outputs. Ensure that required products are installed and that EEGLAB, SPM12, and FieldTrip are on the MATLAB path.

Why do different menus show different subject counts for the same study?

This typically happens when one menu computes counts from the currently loaded dataset structs while another menu uses the study PDI snapshot. The correct source of truth for a study is the resolved PDI, including internal controls. Update plotting menus to compute denominators from PDI membership rather than dataset lists.

I get table assignment errors during testing or plotting. What is the likely cause?

A common cause is an uninitialized or empty struct entry appended to a PDI or results array. Downstream code often assumes each entry has required fields (for example patient_code). Fix the writer to filter out entries missing required fields before saving.

Quick diagnostic

Inspect the saved PDI or results file and confirm there are no trailing entries with empty fields. If present, the cohort builder or results aggregator should be hardened.

I installed EEGLAB but which('eeglab') says not found. Why?

EEGLAB must be added to the MATLAB path. Installing it on disk is not sufficient. Add the EEGLAB folder (the one containing eeglab.m) to the path and save the path. If the toolbox performs dependency checks at startup, rerun after the path update.

My MEG variance is unusually low. Is that expected?

Low variance can indicate correct unit scaling (for example femtotesla), but it can also indicate an accidental unit mismatch. Verify the unit convention used by your loader and preprocessing pipeline, and ensure consistent units across subjects before pooling. Unit mistakes can dominate energy metrics and distort coupling proxies.

More help

If an error message includes the function and line number, start by checking for inconsistent struct sizes, missing fields, or empty cohort entries. These are the most common root causes in population workflows.
