Preprocessing

Preprocessing converts raw neurophysiological recordings into a standardized representation suitable for downstream energy, coupling, and entropy-based analysis. In the BEI Toolbox, preprocessing is designed to be deterministic, unit-consistent, and label-aligned. The output of preprocessing is a standardized subject-level patient_data struct saved as a MATLAB .mat file for each subject. During preprocessing, the toolbox writes these files into a mat-files folder under the preprocessing output directory (typically the dataset PDI directory). These subject files are the stable computational input for downstream plotting, study assembly, and energy-metrics computation.

Scientific principles

  • Signal integrity: remove or attenuate artifacts that introduce non-neural structure into energy distributions.
  • Comparable units: enforce consistent units within a modality (microvolts for EEG, consistent convention for MEG).
  • Label alignment: channel labels or ROI names must match the time series columns and be handled explicitly if missing.
  • Provenance: record the preprocessing recipe, toolbox version, and key parameters.
Determinism in practice

If your pipeline uses stochastic components (for example certain ICA initializations), fix random seeds and store algorithm configurations. Prefer deterministic alternatives where feasible.

Pipeline structure

  1. Discovery: locate subject files and resolve identifiers.
  2. Loading: import raw signals using modality-specific readers (EEGLAB, FieldTrip, SPM).
  3. Cleaning: filtering, artifact correction, and bad channel handling.
  4. Standardization: resampling if required, unit enforcement, label mapping, and formatting.
  5. Persistence: write a PDI file containing patient_data with provenance metadata.

EEG preprocessing

  • Bandpass filtering consistent with the analysis band of interest.
  • Notch filtering for line noise when appropriate for the acquisition environment.
  • Bad channel identification and a recorded interpolation strategy if used.
  • Artifact correction, commonly ICA-based, with explicit component rejection criteria.
Flat or low-amplitude segments

Some EEG recordings contain initial calibration or impedance-check intervals with near-flat signals. These segments can bias energy distributions toward low-energy states and should be detected and removed using objective criteria before downstream feature extraction.

MEG preprocessing

  • Bad channel exclusion and consistent sensor selection.
  • Filtering and optional resampling to match analysis assumptions.
  • ICA and automated artifact labeling when rank supports decomposition.
  • Unit scaling enforcement to avoid artificial variance compression or inflation.

The MEG pipeline should explicitly handle the corner case where data rank is zero or too low for ICA. In such cases, skip ICA and proceed with conservative filtering and robust checks.

fMRI preprocessing

  • Coregistration and spatial normalization to standard space consistent with the chosen atlas.
  • Temporal filtering consistent with resting-state assumptions and the sampling rate (TR).
  • ROI extraction using a fixed atlas with explicit handling of missing ROIs.
  • Output formatting as a [T x R] matrix with ROI labels aligned to columns.

Outputs and provenance

Each preprocessing run writes a PDI file that includes at minimum:

  • Preprocessed time series (for example eegarray, fMRIarray, or megarray) and channel or ROI labels
  • Subject identifier, diagnosis label, modality, dataset name
  • Toolbox version and preprocessing parameter snapshot
  • Optional intermediate outputs needed to reproduce derived features
